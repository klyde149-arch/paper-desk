# LIVE-контур C3b на реальном счёте Т-Инвестиций (T-Invest API v2)

Дата: 2026-07-15. Статус: **Phase 0 реализована** (код+тесты без токена). Стратегия C3b **не меняется** —
меняется только слой исполнения. Paper-эталон: `tools/rf_engine.ps1` (профиль c3b, форвард с 2026-07-10).

## Решения пользователя (зафиксированы 2026-07-15)
- Капитал **~700 000 ₽**, ОСНОВНОЙ счёт, свободные деньги. Фьючерсы на счёте — эксклюзив бота;
  свои акции пользователь держать может (сверка «bot-owned lots»).
- Live = **полный C3b**: рукава B+A (фьючерсы BR NG GOLD SILV Si RTS CNY MIX) + M (акции TQBR).
- Хостинг: тот же VPS; первым шагом Phase 1 — probe доступности API с VPS.
- Аварийный стоп: **−35% от пика** → закрыть всё + `data/HALT_RF_LIVE` + ручной разбор.
  ⚠ Бэктест-DD C3b 40–44% — халт возможен в штатной просадке, это осознанный компромисс.

## Ключевые дизайн-решения (нейтрализуют главные точки провала)
1. **Polling-first, без стримов.** Каждый минутный тик = полная сверка (портфель + заявки +
   стоп-заявки + маржа, 4–6 unary — в ~10 раз внутри rate-лимитов). «Обрыв стрима без
   восстановления состояния» исчезает как класс: обрыв сети = прерванный oneshot-тик, следующий
   тик всё чинит. Стримы — опциональный ускоритель v2, источником истины не становятся.
2. **Брокерские стоп-заявки (StopOrdersService)**: переживают падение бота/VPS/сети и ночной гэп
   00:30→10:00; хранятся у брокера, не на бирже → перевыставления не жгут лимит транзакций Мосбиржи.
3. **Market-входы в окно 10:01–10:15**: paper исполняет вход по open±SLIP 3bps — спред ликвидных
   фронтов сопоставим. Нет зависших лимиток → флуд-сборы исключены (ratio ~1:1.5 при лимите 10:1).
4. **Латентность не критична by design**: C3b дневная, вход на открытии. Метрики
   t_signal→t_post→t_fill логируются per-заявка (latency_log.csv + trades.json).
5. **Сигналы байт-в-байт = paper**: общий код `tools/lib_rf_signals.ps1` (вынос защищён
   golden-replay), те же данные MOEX ISS, те же непрерывные серии (свой каталог
   `data/live_rf/series`, при старте сидируется копией paper-серий).

## Тик-пайплайн (каждую минуту)
lock → kill-файлы (`HALT`, `HALT_RF_LIVE`, `HALT_RF_CLOSE`=закрыть всё, `HALT_RF_ENTRIES`) →
weekend-гейт (внебиржевые выходные НЕ торгуем) → preflight (GetMarginAttributes; сбой = тик прерван,
вотермарки на месте, алерт на 5-м подряд) → **сверка**: снимок стоп-заявок → TP1-sync (до D5!) →
reconcile → state-machine polling → MTM → governors → расписание MSK:
- **00:20** дневной хук (бар D финален в 00:15): докатка серий + SHA-мысль о кросс-сверке с paper,
  фронты (ISS) + сверка экспираций с API, сигналы Donchian/setA → entry-интенты (исполнение завтра
  10:01), детект роллов (≤4 дн до LASTTRADEDATE), **трейл-люстра — ТОЛЬКО здесь** (MFE копится по
  часовикам, стоп двигается раз в день — ровно как paper), EMA20-выходы → exit-интенты, mom-сигнал
  1-го торгового дня;
- **09:45** обновление кэша инструментов/ГО + отложенные stop_replace; **10:01–10:15** exits → entries
  (сайзинг + предиктивный ГО-чек; после fill — стоп-заявка В ТОМ ЖЕ тике); **10:05–18:00** роллы;
  **10:10** mom sells → (после подтверждения) buys; **ежечасно** MFE-трекинг + BE-эмуляция TP1 для
  lots==1 (+эмуляция стопов в sandbox); **каждые 15 мин** MTM-снапшот; **19:30** дневной отчёт
  (латентность, ratio заявки:сделки, пик ГО, дрифты, qty0-пропуски) + Telegram.
- Клиринговые паузы (13:58–14:06, 18:43–19:06, 23:48–00:32 — уточнить в Phase 1): никаких постановок.

## State machine заявки
`INTENT →(persist ДО post)→ POSTED → FILLED | PARTIAL(стоп на filled_lots сразу; добор ≤3 попыток) |
REJECTED(без ретрая; roll_open REJECTED = карточка закрывается reason=roll-fail) |
LOST(adopt по операциям / repost ТЕМ ЖЕ order_id ≤3) | EXPIRED(entry живёт только свой день)`.
Терминальные интенты убираются в том же тике. Предохранитель: >20 постановок/день → entries_halt.
postOrder ≠ факт исполнения: подтверждение только через GetOrderState + GetOperations.

## Reconcile (каждый тик; в dryrun — log-only)
Фьючерсы (строго, эксклюзив бота): **D2** чужая позиция → аварийное закрытие + halt; **D4** карточка
без позиции → подтвердить срабатывание стопа по operations (штатно) или карантин + halt; **D5** лоты
≠ (не объяснимо intents/TP1) → усечь к брокеру; **D6** позиция без стоп-заявки → перевзвод немедленно,
2-я подряд неудача → аварийное закрытие. Акции: real ≥ bot — норма; real < bot → усечь бот-леджер +
алерт; продажи только ≤ bot_lots.

## Сайзинг и governors
- `rub_per_pt = min_price_increment_amount / min_price_increment`;
  `lots = floor(riskRub / (stopDist_pts × rub_per_pt))`, зеркало кэпа MAXLEV=3 по нотионалу,
  lots==0 → SKIP qty0 (лог+счётчик). setA stopDist — от фактического филла (max(свинг, 1×ATR)).
- Sleeve-леджеры: core 700k, setA 700k, **mom = 700k×0.5 = 350k и инвестируется целиком**
  (эквивалент paper: полный рукав с весом 0.5 в профиле; половинить и снова взвешивать — двойное
  занижение). Профиль — помесячно-аддитивно как paper.
- ГО: вход ⇔ go_after ≤ 60% × (base − стоимость bot-акций − резерв 50k); мониторинг каждый тик;
  >60% → entries_halt; >75% → LIFO-закрытие. Paper-идентично: day-halt рукава −6% (снимает только
  входы), MAXCONC=3, 1 позиция/символ, re-arm. Дополнительно: профиль-день −8% → entries_halt;
  hard −35% от пика → закрыть всё + HALT_RF_LIVE.

## Роллы
Триггер как paper (≤4 дн до LASTTRADEDATE или биржа сменила фронт; рескейл серии тем же кодом —
иначе разойдутся сигналы). Исполнение в окно 10:05–18:00: cancel стопа → roll_close market → FILLED →
roll_open market (лоты = сохранение нотионала, мин 1) → новый стоп. Новый сигнал в зоне ролла →
вход сразу в следующий контракт (задокументированная исполнительская дивергенция). Никогда не
открывать контракт с ≤4 дн до LT.

## Режимы (env TINVEST_MODE)
- **dryrun** — мутирующие вызовы перехватываются ДО любого транспорта (WOULD CALL в
  dryrun_calls.log), исполнение симулируется по референс-цене, reconcile log-only;
- **sandbox** — песочница; StopOrders там нет → `config.json: {"emulate_stops": true}` (бот-сайд по
  часовикам, код общий с BE-логикой lots==1); заявки песочницы исполняются мгновенно по последней цене;
- **prod** — реальные заявки. Оверрайды фаз — `data/live_rf/config.json` (whitelist,
  max_lots_override, mom_enabled и т.д.), без правки кода.

## Осознанные компромиссы (меряются недельной декомпозицией дивергенции live vs paper futC3b)
- Целые лоты: эффективный риск ≤ paper (floor); TP1 при lots==1 → только BE-перенос.
- Вход в следующий контракт в зоне ролла (paper: вход во фронт + платный ролл назавтра).
- Брокерский stop-market real-time vs paper «касание часовика» — направленно нейтрально.
- Комиссии: до Phase 1 модель 1bps (fee_est) — заменить фактом из operations/тарифа.
- Hard-halt −35% внутри бэктест-DD 40–44% — выбор пользователя.
