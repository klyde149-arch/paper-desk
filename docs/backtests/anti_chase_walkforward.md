## Протокол (предрегистрация)

- Данные: `data\deep` (Bybit linear 4h + реальный funding, BTC с 2020-03, обновлено до 2026-07-14), F&G с 2018.
- Базовый конфиг = live-аналог: `BtcFilter + MaxAtrPct 3 + FlatMode skip + FundingDir deep + FundingFilter`.
- Каждый календарный год: **dev = первые 70%** доступного окна, **reserve = последние 30%** (не участвует в выборе).
- Атрибуция сделки окну — по дню ВХОДА (фильтры действуют на входе). Агрегатор: `tools\analyze_wf_years.ps1` (резерв печатает только с `-Final`).
- Добавления по одному: сетка ext ∈ {1.0, 1.25, 1.5} × rsi ∈ {60, 65}, отбор на dev всех лет, затем ОДНО вскрытие резерва.
- Критерий финала (зафиксирован до вскрытия): принять, если на резервах ΔΣR ≥ 0 в большинстве лет И суммарно > 0.

## Интерпретация

1. Прямой эффект гейта крошечный: он отсекает ~13 входов за 6 лет (~2/год). Косвенный эффект
   огромный: освободившийся слот MaxConcurrent занимает другая сделка, меняется equity-путь и
   сайзинг всех последующих сделок — «каскад» перетасовывает сотни сделок. На таком отношении
   сигнал/шум dev-выигрыш +18.9R оказался отбором удачного шума (я выбрал ext1.25 именно за
   лучший dev → регрессия к среднему на резерве, классика).
2. «Горячий карман» из Фазы 0 — честное описание прошлого, но слишком редкое, чтобы запрет
   улучшал портфель системно.
3. Вывод для практики: поздние входы вида 14.07 статистически неприятны, но хирургический
   фильтр против них НЕ доказал пользу. v2 остаётся как есть; возвращаться к теме — после
   существенно большей живой выборки (сейчас закрытых сделок v2 всего 3).
