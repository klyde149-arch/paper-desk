# Предложение: стратегия v2 (ждёт решения пользователя)

**Дата:** 2026-07-09. Основание: [backtest_v2_report.md](backtest_v2_report.md) (6.2 года, медведь+боковик+бык, реальный фандинг). Действующая стратегия: [strategy.md](strategy.md) (не менялась), архив: [strategy_v1_archive.md](strategy_v1_archive.md).

## Предлагаемые изменения (все проверены на 3 режимах рынка)

### 1. FlatMode = skip — НОВОЕ правило
**«BTC во флэте 4h (цена пилит EMA50, EMA20≈EMA50 — режим "range") → новые входы ЗАПРЕЩЕНЫ. Открытые позиции ведутся по обычным правилам.»**
- Живой конфиг: PF 1.22→1.30, +55%→+68% за 6.2 года, DD та же (~13%); улучшение в медведе, боковике и быке.
- halfrisk отклонён по правилу «хуже skip» (проверено: хуже и по итогу, и в слабом периоде).
- Практически: analyze.ps1 уже отдаёт trend_4h BTC; «range» = не входим, ждём тренд.

### 2. Фандинг-фильтр — подтверждение существующего правила
Правило блока C strategy.md («фандинг >+0.05%/8ч — новые лонги запрещены; <−0.05% — шорты») **подтверждено длинной историей** (+27 п.п. к итогу исследовательского конфига, лучше или не хуже во всех периодах). Предложение: перевести из «сентимент-модификаторов» в **обязательные фильтры** входа рядом с фильтром BTC.

### 3. Рекалибровка цели
Заменить «2–4%/мес при просадках 12–25%» на:
**«Трендовые фазы рынка: 1.5–2.5%/мес; полный цикл (вкл. боковики): 0.7–1%/мес при макс. просадке 13–15%. Вязкий боковик (как 2023–24) — около нуля месяцами; это норма, а не поломка.»**
Прошлая цель была откалибрована на удачном 16-месячном окне.

## Отклонено (проверено и не прошло walk-forward по 3 режимам)
| Идея | Причина отказа |
|---|---|
| FlatMode=halfrisk | хуже skip по итогу и в слабом периоде (правило пользователя) |
| Ранний выход при сломе EMA50 | режет прибыль сильнее, чем спасает (медведь +34% vs +59%) |
| Фильтр наклона EMA50 | -80% доходности |
| TP1 = 1.2R / 2.0R | вязкий период PF ≤ 1.00 |
| Флэт-стратегия (отбой от границ при BTC-флэте) | хуже базы во всех 3 периодах — контртренд не окупает стопы |
| Индивидуальные параметры BNB/DOGE | walk-forward провален ранее (оверфит) |

## Как внедряем (если одобришь)
1. strategy.md: добавить правило FlatMode=skip в сетап A; фандинг-фильтр пометить обязательным; заменить абзац цели.
2. Цикл: на каждом тике перед сканом сетапов проверять режим BTC; «range» → «сегодня без новых входов» в журнале.
3. Ожидания/лимиты риска не меняются (0.5–0.75%, ≤2 позиции, −3%/−8%/−15%).

**Ответь «внедряй v2» — внесу; «не внедряй» — торгуем по v1, файл останется предложением.**
